Structural Elements

Element Description
<html>Root element of an HTML document.
<head>Contains metadata and links to external resources.
<body>Contains the content of an HTML document.
<header>Represents introductory content, typically a group of introductory or navigational aids.
<footer>Represents a footer for its nearest sectioning content or sectioning root element.
<section>Represents a standalone section which doesnâ€™t have a more specific semantic element to represent it.
<article>Represents a self-contained composition in a document.
<aside>Represents a portion of a document whose content is only indirectly related to the document's main content.
<nav>Represents a section of a page that links to other pages or to parts within the page: a section with navigation links.
<div>A generic container for flow content that by itself does not represent anything.
<main>Represents the dominant content of the <body> of a document.

Text Content Elements

Element Description
<h1> - <h6>Heading elements, <h1> is the highest (or most important) level and <h6> is the lowest.
<p>Represents a paragraph of text.
<span>A generic inline container for phrasing content, which does not inherently represent anything.
<strong>Represents text of strong importance.
<em>Represents emphasized text, often displayed in italics.
<blockquote>Represents a section that is quoted from another source.
<ul>Represents an unordered list of items.
<ol>Represents an ordered list of items.
<li>Represents an item in a list.
<br>Produces a line break in text (carriage-return).
<hr>Represents a thematic break between paragraphs of a text, usually a horizontal line.
<pre>Represents preformatted text, where whitespace is preserved.
<code>Represents a fragment of computer code.
<mark>Represents text which is highlighted for reference purposes.

Form Elements

Element Description
<form>Represents a document section that contains interactive controls for submitting information.
<input>Represents a typed data field allowing the user to enter data.
<textarea>Represents a multi-line plain-text editing control.
<button>Represents a clickable button.
<select>Represents a control that provides a menu of options.
<option>Represents an option in a <select> element.
<label>Represents a caption for an item in a user interface.
<fieldset>Used to group several controls and labels within a web form.
<legend>Represents a caption for the content of its parent <fieldset>.
<datalist>Represents a list of options for the <input> element.
<output>Represents the result of a calculation or user action.
<progress>Represents the completion progress of a task.
<meter>Represents a scalar measurement within a known range.

Interactive Elements

Element Description
<a>Represents a hyperlink, linking to another resource.
<button>Represents a clickable button.
<details>Represents a disclosure widget from which the user can obtain additional information.
<summary>Represents a summary, legend, or caption for a <details> element.
<dialog>Represents a dialog box or window.
<menu>Represents a list of commands that a user can perform or activate.

Media Elements

Element Description
<img>Embeds an image in the document.
<audio>Embeds sound content in a document.
<video>Embeds a media player which supports video playback.
<picture>Serves as a container for zero or more <source> elements and one <img> element.
<source>Specifies multiple media resources for media elements.
<track>Specifies text tracks for <video> and <audio> elements.
<map>Defines an image map, allowing clickable areas in an image.
<area>Defines a clickable area inside an image map.
<svg>Defines a container for SVG graphics.
<object>Embeds an external resource, such as an image, video, or a browser plug-in.
<embed>Embeds external content, typically multimedia.
<param>Defines parameters for an <object> element.
<iframe>Embeds another HTML page within the current page.

Table Elements

Element Description
<table>Represents a table, used to arrange data in rows and columns.
<tr>Represents a row of cells in a table.
<td>Represents a cell in a table.
<th>Represents a header cell in a table.
<thead>Groups the header content in a table.
<tbody>Groups the body content in a table.
<tfoot>Groups the footer content in a table.
<caption>Represents a title or explanation for the table.
<colgroup>Groups columns in a table for formatting.
<col>Specifies column properties for each column within a <colgroup> element.

Semantic Elements

Element Description
<section>Represents a standalone section in a document, which doesn't have a more specific semantic element to represent it.
<article>Represents a self-contained composition in a document, page, application, or site.
<aside>Represents a portion of a document whose content is only indirectly related to the document's main content.
<header>Represents introductory content, typically a group of introductory or navigational aids.
<footer>Represents a footer for its nearest sectioning content or sectioning root element.
<main>Represents the main content of the <body> of a document.
<nav>Represents a section of a page that links to other pages or to parts within the page: a section with navigation links.
<figure>Represents self-contained content, potentially with an optional caption, which is specified using the <figcaption> element.
<figcaption>Represents a caption or legend for the content of its parent <figure> element.

Metadata Elements

Element Description
<title>Defines the title of the document, shown in the browser's title bar or tab.
<meta>Represents metadata that cannot be represented by other HTML elements.
<link>Specifies relationships between the current document and an external resource, typically used to link to stylesheets.
<style>Contains style information for a document, typically CSS.
<base>Specifies the base URL for relative URLs in the document.
<script>Used to embed or reference executable script.
<noscript>Defines alternative content to be displayed when a script is not executed.
<template>Defines a container for content that should be hidden when the page loads, but can be instantiated later using JavaScript.
<slot>Defines a placeholder inside a Web Component that users can fill with their own markup.
<basefont>Specifies a default font size, color, and face for all text in a document (deprecated).
<bgsound>Specifies a sound to play while a document is being viewed (deprecated).
<command>Defines a command that the user can invoke (not supported by any major browser).
<content>Used in Shadow DOM as a placeholder for unknown content (deprecated).
<dialog>Defines a dialog box or window.
<shadow>Used to encapsulate the style and markup of a custom element (deprecated).
